a 2.1.20
🎨 Новая система кастомизации внешнего вида машин
Добавлена возможность выбора 5 цветов кузова (Зелёный, Красный, Чёрный, Мокрый асфальт, Белый) непосредственно в карточке автомобиля.

Реализована тонировка стёкол – теперь игрок может включать/выключать затемнение через Toggle в интерфейсе.

🛠 Расширение данных CarBlueprint
Добавлены поля:

bodyColor (Color) – хранит выбранный цвет кузова.

hasTint (bool) – флаг наличия тонировки.

tintLevel (float) – уровень затемнения (0..1).

currentPrice (int) – актуальная цена машины (изменяется при улучшении).

Обновлён метод Clone() для корректного копирования новых полей.

💾 Сохранение и загрузка
В классы CarLevelData и CarBlueprintSaveData добавлены поля для цвета и тонировки.

Методы FillSaveData, LoadFromSave, GetCreatedCarsData, LoadCreatedCars в TechManager обновлены для сохранения/восстановления настроек внешнего вида.

🖥 Интерфейс (UIManager)
В каждой карточке машины добавлена компактная панель с цветовыми кнопками и переключателем тонировки.

Реализована визуальная подсветка выбранного цвета (белая рамка).

Исправлены ошибки компиляции:

Заменены несуществующие свойства borderWidth и borderColor на корректные по-сторонние (borderTopWidth, borderBottomWidth и т.д.).

Добавлен default в switch для сложности, устранено предупреждение CS8524.

🔧 Улучшение и производство машин
В TechManager.UpgradeCar() теперь корректно устанавливается префаб следующего уровня (car.levelPrefabs), что меняет внешний вид улучшенной версии.

При создании новой машины через ProductionManager теперь применяются сохранённые настройки цвета (вызов CarCustomizer.ApplyFromBlueprint).

✅ Итог
Версия 2.1.20 включает полноценную систему покраски и тонировки автомобилей, интегрированную с сохранениями и улучшениями. Все ошибки компиляции устранены, интерфейс расширен, а логика работы с префабами при улучшении исправлена.
